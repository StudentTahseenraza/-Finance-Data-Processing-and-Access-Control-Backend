📖 How to Test APIs

1 Open Swagger Documentation

Click the button below to open interactive API documentation

🚀 Open Swagger UI
2 Authorize / Login

In Swagger UI, click the Authorize button and login with one of the test accounts above.

First, call the POST /auth/login endpoint, then copy the accessToken from response.

3 Set Token for Authentication

After getting the token, click Authorize button and enter: Bearer YOUR_TOKEN_HERE

4 Test Different Roles

Login with different roles to see different permissions:

  • Admin - Can see all endpoints including user management
  • Analyst - Can create, update, delete records
  • Viewer - Can only view data, cannot modify

🧪 API Testing Flow

  1. Authentication - Call POST /auth/login to get token
  2. Create Records - Use POST /records to add transactions
  3. View Records - Use GET /records with filters
  4. Dashboard - Check GET /dashboard/summary for overview
  5. Analytics - Use GET /dashboard/trends for charts
  6. Update/Delete - Modify or remove records
  7. User Management - Admin only: GET /users

🎯 Sample API Calls

Login as Analyst:

curl -X POST https://your-api.com/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "email": "analyst@finance.com",
    "password": "Analyst@123456"
  }'

Create an Expense Record:

curl -X POST https://your-api.com/api/v1/records \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "amount": 150.00,
    "type": "expense",
    "category": "Groceries",
    "date": "2024-04-04",
    "description": "Weekly grocery shopping"
  }'

Get Dashboard Summary:

curl -X GET "https://your-api.com/api/v1/dashboard/summary?start_date=2024-04-01&end_date=2024-04-30" \
  -H "Authorization: Bearer YOUR_TOKEN"

✅ What to Test as HR Reviewer

  • ✅ Authentication - Login/Register functionality
  • ✅ Role-Based Access - Verify different permissions for Admin/Analyst/Viewer
  • ✅ CRUD Operations - Create, Read, Update, Delete records
  • ✅ Filtering & Search - Test all filter parameters
  • ✅ Dashboard Analytics - Check summary, trends, insights
  • ✅ Pagination - Test page and limit parameters
  • ✅ Error Handling - Try invalid inputs to see error responses
  • ✅ Security - Ensure unauthorized access is blocked
