📝 Step-by-Step Testing Guide

1 Open Swagger Documentation

Click the "Open Swagger UI" button above or visit /api-docs endpoint.

2 Login to Get Token

Use the POST /auth/login endpoint with any test account:

{
  "email": "analyst@finance.com",
  "password": "Analyst@123456"
}
3 Authorize in Swagger

Click the "Authorize" button and enter: Bearer YOUR_ACCESS_TOKEN

4 Test Endpoints by Role

Try different endpoints with different roles to verify RBAC:

  • Analyst: Test POST, PUT, DELETE on /records
  • Viewer: Try the same operations - they should fail with 403
  • Admin: Test GET /users and user management
5 Test Filtering and Pagination

Use query parameters on GET /records:

?type=income&start_date=2024-04-01&end_date=2024-04-30&page=1&limit=10
6 Test Analytics

Check dashboard endpoints for summaries and insights:

  • GET /dashboard/summary - Overview stats
  • GET /dashboard/trends - Monthly trends
  • GET /dashboard/insights - Smart insights

🚀 Sample API Calls for Quick Testing

Login as Analyst:

curl -X POST https://finance-backend-api.vercel.app/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "email": "analyst@finance.com",
    "password": "Analyst@123456"
  }'

Create Expense Record:

curl -X POST https://finance-backend-api.vercel.app/api/v1/records \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "amount": 150.00,
    "type": "expense",
    "category": "Groceries",
    "date": "2024-04-04",
    "description": "Weekly shopping"
  }'

Get Dashboard with Filters:

curl -X GET "https://finance-backend-api.vercel.app/api/v1/dashboard/summary?start_date=2024-04-01&end_date=2024-04-30" \
  -H "Authorization: Bearer YOUR_TOKEN"
